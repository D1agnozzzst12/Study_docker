1. docker ps -aq -f staus=running - Контейнеры, которые в данный момент работают.

2. docker ps -aq -f status=paused - Контейнеры которые приастановленны.

3. docker ps -aq -f status=exited - Контейнеры, которые завершили свою работу.

4. docker ps -aq -f status=dead - Контейнеры, которые находятся в состоянии "мертвый".

5. docker ps -aq -f status=created - Контейнеры, которые были созданы, но еще не запущены.



Пример использования:

Чтобы получить идентификаторы всех контейнеров которые запущены вы можете испльзовать статус running:
	docker ps -aq -f status=running





Коды выхода (exit codes) контейнеров Docker могут помочь определить, как завершился процесс внутри контейнера. Вот основные коды выхода, которые вы можете встретить:

1. 0 - Успешное завершение. Программа завершилась без ошибок.

2. 1 - Общая ошибка. Обычно указывает на то, что произошла ошибка выполнения.

3. 2 - Неверный аргумент. Программа была вызвана с неправильными параметрами.

4. 126 - Команда не может быть выполнена. Обычно это озночает, что файл не является исполняемым.

5. 127 - Команда не найдена. Указывает на то, что указанная команда не существуют.

6. 137 - Контейнер был принудительно завершен (Обыно с помощью SIGKILL).

7. 139 - Ошибка с сегментации (segmentation fault). Программа попыталась получить доступ к недопустимой области памяти.

8. 143 - Контейнер был запущен с помощью SIGTERM.


Что бы проверить код выхода последнего завершенного контейнера вы можете использовать:

docker ps -aq -f exited=127






Для остановки всех работающих контейнеров:

docker stop $(docker ps -q)
